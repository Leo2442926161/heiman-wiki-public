# Anwendungstipps

## Benutzerdefinierter Treiber

Heimann-Geräte sind in Hubitat für die Kompatibilität mit den folgenden Treibern optimiert:

| Gerätetyp | Empfohlener Treiber |
|----------|----------|
| Rauch-/CO-Melder | Generischer Zigbee-Rauch-/CO-Sensor |
| Infrarotsensor für den menschlichen Körper | Heiman Bewegungssensor (Drittanbieter) |
| Türmagnetsensor | Generischer Zigbee-Kontaktsensor |
| Wasserlecksensor | Allgemeiner Zigbee-Wasserlecksensor |
| Temperatur- und Feuchtigkeitssensor | Heiman Temperatur-/Feuchtigkeitssensor |

## Zigbee-Optimierung

- Die Modelle Hubitat C-8 und höher verwenden den EFR32-Chip, der die beste Kompatibilität bietet
- Sehen Sie sich den Kanal mit der geringsten Störung an und wählen Sie ihn unter Einstellungen → Zigbee aus
- Verwenden Sie Zigbee-Sniffing-Tools wie Ubisys, um abgebrochene Anrufe zu diagnostizieren

## Regelaufforderungen

- Es wird empfohlen, in Rule Machine die Bedingung „Anti-Fehlalarm-Verzögerung“ (0,5–2 Sekunden) hinzuzufügen
- Aktivieren Sie „Health Check“-Benachrichtigungen für wichtige Regeln, um sicherzustellen, dass sie weiterhin ausgeführt werden
- Organisieren Sie komplexe Automatisierungen im „Sub-Rule“-Modus für einfaches Debuggen und Warten

## Z-Wave-Koexistenz

Wenn Sie Z-Wave-Geräte gleichzeitig nutzen möchten, achten Sie darauf, den Frequenzabstand zwischen Zigbee- und Z-Wave-Kanälen einzuhalten.
