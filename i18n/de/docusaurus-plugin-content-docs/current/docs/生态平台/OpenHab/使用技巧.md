# Anwendungstipps

## Dingkonfiguration

Heiman-Gerät über MQTT-Bindung hinzufügen:

„Java
Sache mqtt:topic:heiman_smoke „Heiman Rauchmelder“ @ „Wohnzimmer“ {
    Kanäle:
        Geben Sie binär ein: Smoke „smoke state“ [ stateTopic="zigbee2mqtt/heiman_smoke", transformationPattern="JSONPATH:$.smoke" ]
        Typnummer: Batterie „Batterieleistung“ [ stateTopic="zigbee2mqtt/heiman_smoke", transformationPattern="JSONPATH:$.battery" ]
}
„

## Artikeldefinition

„Java
Kontaktieren Sie Smoke_Alarm „Rauchalarm [MAP(smoke.map):%s]“ {channel="mqtt:topic:heiman_smoke:smoke" }
Number Smoke_Battery „Batterie des Rauchmelders [%d%%]“ {channel="mqtt:topic:heiman_smoke:battery" }
„

## Koordinatenauswahl

Es wird empfohlen, einen USB-Koordinator mit **CC2531**- oder **CC2652**-Chip zu verwenden und sicherzustellen, dass die Firmware-Version die neueste ist.

## Debugging-Tipps

- Sehen Sie sich den „Online/Offline“-Status des Dings in der Paper-Benutzeroberfläche an
- Verwenden Sie „openhab-cli info“, um zu überprüfen, ob die Bindungen korrekt geladen wurden
- Suchen Sie im Protokoll nach dem Schlüsselwort „heiman“, um Probleme mit der Gerätekommunikation zu beheben
